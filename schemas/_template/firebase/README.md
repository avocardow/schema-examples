# Firebase / Firestore Format ŌĆö Implementation Guide

## Quick Reference

| Aspect | Convention |
| ------ | ---------- |
| Language | JavaScript |
| File extension | `.js` |
| Module system | ES modules (`import` / `export`) |
| Field naming | `camelCase` |
| Collection naming | `camelCase` or `snake_case` (your choice, be consistent) |
| ID type | Auto-generated by Firestore |
| Timestamps | `Timestamp.now()` from `firebase/firestore` |

## Key Rules

1. **ES modules** ŌĆö Use `import` / `export`, not CommonJS `require`.
2. **Three exports per file**: JSDoc `@typedef`, `createXxx` factory, `xxxConverter`.
3. **No `id` in factory output** ŌĆö Firestore assigns the document ID. The converter's `fromFirestore` reads `snapshot.id`.
4. **Timestamps** ŌĆö `import { Timestamp } from "firebase/firestore"`. Use `Timestamp.now()` in factories.
5. **Nullable fields** ŌĆö Use `?? null` in both the factory and converter. **Non-nullable fields** use bare `data.fieldName` in the converter (no `?? default` fallbacks). The factory CAN use `?? default` for optional parameters, but the converter must trust the stored data.
6. **FK annotations** ŌĆö In the JSDoc `@typedef`, annotate foreign key fields with `- FK ŌåÆ collection_name`:
   ```
   * @property {string} userId - FK ŌåÆ users
   ```
7. **Enums** ŌĆö `export const ENUM_NAME = /** @type {const} */ ({...})`.

## File Template

```javascript
// {table_name}: Brief description of the table.
// See README.md for full design rationale.

import { Timestamp } from "firebase/firestore";

export const TABLE_STATUS = /** @type {const} */ ({
  ACTIVE: "active",
  INACTIVE: "inactive",
});

/**
 * @typedef {Object} TableNameDocument
 * @property {string} id
 * @property {string} userId - FK ŌåÆ users
 * @property {string} name
 * @property {string|null} description
 * @property {typeof TABLE_STATUS[keyof typeof TABLE_STATUS]} status
 * @property {Object|null} metadata
 * @property {import("firebase/firestore").Timestamp} createdAt
 * @property {import("firebase/firestore").Timestamp} updatedAt
 */

/**
 * @param {Omit<TableNameDocument, "id" | "createdAt" | "updatedAt">} fields
 * @returns {Omit<TableNameDocument, "id">}
 */
export function createTableName(fields) {
  return {
    userId:      fields.userId,
    name:        fields.name,
    description: fields.description ?? null,
    status:      fields.status ?? TABLE_STATUS.ACTIVE,
    metadata:    fields.metadata ?? null,
    createdAt:   Timestamp.now(),
    updatedAt:   Timestamp.now(),
  };
}

export const tableNameConverter = {
  /** @param {Omit<TableNameDocument, "id">} doc */
  toFirestore(doc) {
    return { ...doc };
  },
  /** @param {import("firebase/firestore").QueryDocumentSnapshot} snapshot */
  fromFirestore(snapshot, options) {
    const data = snapshot.data(options);
    return {
      id:          snapshot.id,
      userId:      data.userId,
      name:        data.name,
      description: data.description ?? null,
      status:      data.status,
      metadata:    data.metadata ?? null,
      createdAt:   data.createdAt,
      updatedAt:   data.updatedAt,
    };
  },
};
```

## Suggested Indexes

Firestore indexes are defined in `firestore.indexes.json`, not in the schema files. Add a comment block at the end of each file suggesting which indexes to create:

```javascript
/*
 * Suggested Firestore indexes:
 *   collection: "table_name"
 *   - userId ASC, createdAt DESC  (query user's items by date)
 *   - status ASC, createdAt DESC  (filter by status)
 */
```

## Field Type Mapping

| Pseudo Code | Firestore / JSDoc Type |
| ----------- | ---------------------- |
| `string` | `string` |
| `integer` | `number` |
| `bigint` | `number` |
| `boolean` | `boolean` |
| `decimal` | `number` |
| `uuid` (FK) | `string` (store the ID as a string) |
| `json` | `Object` |
| `timestamp` | `Timestamp` (from `firebase/firestore`) |
| `enum(...)` | `string` with const enum export |
| `text[]` | `string[]` |

## Gotchas

- **No enforced foreign keys** ŌĆö Store referenced document IDs as strings. Referential integrity is your responsibility. Annotate FK fields in the JSDoc `@typedef` with `- FK ŌåÆ collection_name`.
- **No `id` in the factory** ŌĆö The document ID is set by Firestore when you call `addDoc` or `setDoc`.
- **`?? null`** ŌĆö Use this for nullable fields in both factory and converter. **Do NOT use `?? default` on non-nullable fields in the converter** ŌĆö the converter trusts stored data and uses bare `data.fieldName`. The factory CAN use `?? default` for optional constructor parameters.
- **Converter's `fromFirestore`** ŌĆö Always include `id: snapshot.id` to attach the document ID.
- **Timestamps are Firestore `Timestamp` objects**, not JS `Date`. Use `Timestamp.now()` in factories.
- **No `updatedAt` auto-update** ŌĆö Firestore doesn't auto-update timestamps. Handle in your write logic with `serverTimestamp()`.
- **Enum const objects** ŌĆö Use `/** @type {const} */` for type narrowing in JSDoc-typed codebases.
